### ğŸ›  Technical Deep Dive

#### 1. Pathfinding via Dijkstraâ€™s Algorithm
The core of the navigation engine is a customized **Dijkstraâ€™s Algorithm**. 
* **Multilayer Logic:** We implemented the algorithm to handle pathfinding separately for each floor and then combined them through "connection nodes" (stairs/lifts).
* **Data Structure:** Instead of a traditional database, we utilized a highly optimized **2D Array Matrix** to store room information and connectivity, ensuring lightning-fast path calculations within the browser.

#### 2. 3D Visualization with p5.js & Blender
To create the 3D experience without the overhead of a heavy game engine:
* **Hybrid Modeling:** We used **Blender** to assist in the initial spatial design and **p5.js (Processing)** for the real-time web rendering.
* **Interactive UX:** Features like "Transparent Walls" and "Floor Switching" were coded in JavaScript to allow users to see through the building structure and focus on their specific path.

#### 3. Cloud-Native Deployment (IBM Cloud & Kubernetes)
This project was a deep dive into modern DevOps practices:
* **Containerization:** The entire application was containerized using **Docker**.
* **Orchestration:** We deployed the app on **IBM Cloud** using **Kubernetes**, ensuring high availability and professional-grade hosting.

#### 4. Frontend Foundation: HTML5, CSS3, & JavaScript
As a key contributor to the frontend, I built the application's core structure using standard web technologies:
* **HTML5 (Index):** Structured the single-page application (SPA) environment to host the 3D canvas and user control panels.
* **CSS3:** Designed a responsive and clean user interface, ensuring that the navigation controls and search bars were intuitive and "spoke for themselves."
* **JavaScript (ES6+):** Acted as the "brain" of the frontend, handling real-time user inputs, managing state changes between different floors, and bridging the UI with the 3D rendering engine.
